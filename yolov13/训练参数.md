## 推荐训练参数

### 方案一：平衡型（推荐首次训练）

适用于：需要平衡训练速度和模型性能的常规训练

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 模型 | `yolov13n.pt` | 轻量级模型，适合快速验证 |
| 批量大小 (batch) | **32-64** | RTX 5090 显存充足，可大幅提升 |
| 输入图片大小 (img) | **1280** | 更大分辨率有利于检测小人脸 |
| 训练轮数 (epochs) | **100-300** | WiderFace 官方常用 300 轮 |
| 优化器 | SGD | 稳定收敛 |
| 初始学习率 (lr0) | 0.01 | 标准学习率 |
| 学习率衰减 (lrf) | 0.01 | 最终学习率为 lr0 * lrf |
| 动量 (momentum) | 0.937 | 标准值 |
| 权重衰减 (weight_decay) | 0.0005 | 标准值 |
| 预热轮数 (warmup_epochs) | 3 | 标准值 |
| 数据加载线程 (workers) | **16** | 充分利用多核CPU |
| GPU编号 | 0 | 单卡训练 |

```bash
python train_widerface.py \
    --model yolov13n.pt \
    --epochs 300 \
    --batch 64 \
    --img 1280 \
    --device 0
```

## 早停机制说明

YOLOv13 内置了完善的早停（Early Stopping）机制：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `patience` | 100 | 连续多少轮无改善则停止训练 |
| 监控指标 | mAP@0.5 | fitness = mAP |
| 保存最佳 | best.pt | 自动保存最佳权重 |

### 早停触发条件
```
当前epoch的mAP >= 历史最佳mAP → 更新最佳epoch
如果 (当前epoch - 最佳epoch) >= patience → 停止训练
```

### 如何调整早停
```python
# 在 train_widerface.py 中添加
patience=50,    # 更快停止（保守训练）
patience=200,   # 更慢停止（激进训练）
patience=0,     # 禁用早停
```

## 注意事项

1. **如果显存不足**：减小 `batch` 或 `img` 参数
2. **如果训练过慢**：减小 `img` 参数或增加 `workers`
3. **小人脸检测**：优先使用方案三，增加输入分辨率
4. **多卡训练**：设置 `device='0,1,2,3'` 使用多卡
5. **早停正常**：看到 "Training stopped early" 是正常现象，说明模型已收敛
